Introduction

The sensor-based push-up and sit-up counter system is designed to accurately count and analyze physical movements during workouts. This system utilizes sensors attached to the user's body and sends data to a central server for real-time feedback and tracking.

The system includes two main parts: the client-side devices attached to the user (for push-ups and sit-ups) and a server that receives and processes the data. The sensors used are MPU-based, which detect orientation and movement, while the communication is handled using BLE (Bluetooth Low Energy).

  • The primary objective of the system is to help users track their workout performance accurately, ensuring that each exercise is performed correctly.
  • The system also provides feedback to the user regarding their posture and execution, promoting proper form and reducing the risk of injury.
  • Data collected from the sensors is stored and can be reviewed for progress tracking and statistical analysis.
Features

The following are the features of the Sensor-Based Push-up and Sit-up Counter System:

  • Connects to the ESP32 server via BLE.
  • Calculates correct and incorrect push-up/sit-up movements.
  • Sends feedback to the ESP32 server based on the movements.
  • Saves the results (name, mode, correct/incorrect movement, time) into a CSV file.
  • Resets the ESP32 server and deletes the CSV data.

Installation

To ensure the proper functioning of the project, it's essential to first install the necessary prerequisites and follow the setup steps carefully. This will enable all components to work seamlessly together. Below are the detailed instructions to help you set up the system:.

  1. Clone the repository:
    2. git clone https://github.com/AndriSatia/Tugas_Akhir
  2. Create a virtual environment and activate it:
    3. python -m venv env source env/bin/activate
  3. Install dependencies:
    4. pip install django pandas bleak
  4. Migrate database:
    5. python manage.py migrate
  5. Run the server:
    6. python manage.py runserver
How to Use

Start the Server

To run the Django server, use the following command:

python manage.py runserver

Then, open your browser and access the interface at:

http://localhost:8000

Pages

  • Connect BLE: Connect to the ESP32 server via Bluetooth.
  • Push-up Counter: Start counting push-ups.
  • Sit-up Counter: Start counting sit-ups.
  • History: View stored movement data in CSV.

Data Management

All movement data (correct/incorrect) is saved in the data/history.csv file. You can:

  • Reset ESP32: Reset the ESP32 server via the interface.
  • Clear CSV: Delete all CSV data while retaining the header.

API Endpoints

  • /connect_ble_view: Connect to the ESP32 device via BLE.
  • /get_counter: Get the current count of push-ups or sit-ups.
  • /reset_esp32_view: Reset the ESP32 server.
  • /add_data_to_csv: Add new data to the CSV file.
  • /clear_csv: Clear all data in the CSV, keeping the header.
Important Notes

This project is designed for internal use only and should only be edited by registered contributors. Please ensure the following key points are adhered to when using and modifying the system:

BLE Configuration

The system connects to an ESP32 device using the MAC address:

34:B7:DA:F8:88:FA

You can change it according to your ESP32's address.

The project relies on Bluetooth Low Energy (BLE) for communication. Make sure that your ESP32 is configured correctly and that the UUIDs for the BLE characteristics match between the ESP32 device and the code.

UUID: your-uuid-here

Installation Requirements

Ensure that the bleak library is installed for BLE functionality:

pip install bleak

If you encounter any issues with the installation of bleak, use the following command:

pip install https://github.com/hbldh/bleak/archive/refs/heads/develop.zip

Device Configuration

Before running the project, double-check that your BLE device is properly configured. Improper configurations may cause connection issues or unreliable performance.
